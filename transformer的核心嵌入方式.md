# Transformer 核心嵌入方式

## 词嵌入（Token Embedding）和位置嵌入（Position Embedding）是 Transformer 的两大核心嵌入，
以红楼梦文本 Transformer 来举例。

## 先给核心结论

| 维度       | 词嵌入 (Token Embedding)                         | 位置嵌入 (Position Embedding)                     |
| ---------- | ------------------------------------------------ | ------------------------------------------------- |
| 核心作用   | 把「字符 / 单词」转化为机器能理解的向量          | 把「字符在序列中的位置」转化为向量                |
| 解决问题   | 让模型知道「这个字符是什么」                    | 让模型知道「这个字符在序列的哪个位置」            |
| 输入依据   | 字符的唯一索引（比如「红」→0，「楼」→1）        | 字符在序列中的位置索引（比如第0位、第1位）        |
| 是否唯一   | 同一字符的词嵌入始终相同（训练中更新）          | 同一位置的位置嵌入始终相同（固定 / 训练）         |

---

## 1. 词嵌入（Token Embedding）

### 通俗理解

词嵌入是给每个字符 / 单词发一个「身份 ID 向量」—— 不管这个字符出现在序列的第 1 位还是第 10 位，它的「身份向量」都是同一个（训练过程中会慢慢优化）。

### 结合代码举例

序列是 `[红, 楼, 梦, ，]`（对应索引 `[0,1,2,3]`），

词嵌入层 `nn.Embedding(vocab_size, n_embd)` 会做：
- 「红」（索引 0）→ 词嵌入向量：`[0.1, 0.2, 0.3, ...]`
- 「楼」（索引 1）→ 词嵌入向量：`[0.4, 0.5, 0.6, ...]`
- 「梦」（索引 2）→ 词嵌入向量：`[0.7, 0.8, 0.9, ...]`
- 「，」（索引 3）→ 词嵌入向量：`[1.0, 1.1, 1.2, ...]`

不管「红」出现在序列第 0 位还是第 10 位，它的词嵌入都是 `[0.1, 0.2, 0.3, ...]`（训练中会更新，但和位置无关）。

### 代码实现（第198行）

```python
tok_emb = self.token_embedding(idx)  # (B, T) → (B, T, n_embd)
```

| 参数 | 值 | 含义 |
|------|--------|------|
| `vocab_size` | 4244 | 词汇表大小（红楼梦中的字符数） |
| `n_embd` | 384 | 每个字符的嵌入向量维度 |

---

## 2. 位置嵌入（Position Embedding）

### 通俗理解

位置嵌入是给每个位置发一个「位置 ID 向量」—— 不管这个位置上是哪个字符，位置向量只和「第几位」有关。

### 结合代码举例

同样是序列 `[红, 楼, 梦, ，]`（长度 4），位置嵌入会做：
- 第 0 位（红）→ 位置嵌入向量：`[0.01, 0.02, 0.03, ...]`
- 第 1 位（楼）→ 位置嵌入向量：`[0.04, 0.05, 0.06, ...]`
- 第 2 位（梦）→ 位置嵌入向量：`[0.07, 0.08, 0.09, ...]`
- 第 3 位（，）→ 位置嵌入向量：`[0.10, 0.11, 0.12, ...]`

如果「红」出现在第 1 位，它的位置嵌入就变成 `[0.04, 0.05, 0.06, ...]`（和位置绑定，和字符无关）。

### 代码实现（第199行）

```python
pos_emb = self.pos_embedding(torch.arange(T, device=device))  # (T) → (T, n_embd)
```

| 参数 | 值 | 含义 |
|------|--------|------|
| `block_size` | 256 | 序列最大长度 |
| `n_embd` | 384 | 位置嵌入维度（与词嵌入一致） |

### 为什么需要位置嵌入？

Transformer 的自注意力层是「无序的」—— 如果只给词嵌入，模型分不清 `[红, 楼, 梦]` 和 `[梦, 楼, 红]` 的区别（因为词嵌入只看字符本身，不看顺序）。

位置嵌入就是用来给模型「标顺序」的。

### 位置编码的两种方式

| 方式 | 说明 | 你的代码 |
|------|------|----------|
| **可学习位置编码** | 位置向量作为可训练参数 | ✅ 用了 `nn.Embedding(block_size, n_embd)` |
| **固定位置编码（Sinusoidal）** | 用 sin/cos 函数直接计算 | ❌ 没使用 |

固定位置编码公式（了解即可）：
```
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
```

---

## 3. 两者的结合（Transformer 的输入）

Transformer 的最终输入 = 词嵌入 + 位置嵌入（逐元素相加），比如：

- 第 0 位「红」的最终输入：`[0.1+0.01, 0.2+0.02, 0.3+0.03, ...] = [0.11, 0.22, 0.33, ...]`
- 第 1 位「楼」的最终输入：`[0.4+0.04, 0.5+0.05, 0.6+0.06, ...] = [0.44, 0.55, 0.66, ...]`
- 第 2 位「梦」的最终输入：`[0.7+0.07, 0.8+0.08, 0.9+0.09, ...] = [0.77, 0.88, 0.99, ...]`
- 第 3 位「，」的最终输入：`[1.0+0.10, 1.1+0.11, 1.2+0.12, ...] = [1.10, 1.21, 1.32, ...]`

这样模型既能知道「这个字符是什么」，又能知道「这个字符在哪个位置」。

### 代码实现（第202行）

```python
x = tok_emb + pos_emb  # (B, T, n_embd) + (T, n_embd) → (B, T, n_embd)
```

**为什么是相加而不是拼接？**
- 相加：维度不变，计算更高效，信息融合
- 拼接：维度翻倍，计算量大
- 实践证明相加效果很好

## 5. 总结

| 概念 | 作用 | 代码位置 |
|------|------|----------|
| 词嵌入 | 让模型理解「字符的含义」 | 第176行: `nn.Embedding(vocab_size, n_embd)` |
| 位置嵌入 | 让模型理解「字符的位置」 | 第178行: `nn.Embedding(block_size, n_embd)` |
| 两者相加 | 融合语义和位置信息 | 第202行: `tok_emb + pos_emb` |

**关键点**：
- 词嵌入是每个字符的「身份」，位置嵌入是每个位置的「门牌号」
- Transformer 本身是「无序」的，必须靠位置嵌入注入顺序信息
- 该代码使用了**可学习的位置嵌入**，训练过程中会自动优化位置向量
- 嵌入维度 `n_embd=384`，确保词嵌入和位置嵌入维度一致才能相加
