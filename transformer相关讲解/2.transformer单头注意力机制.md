# 基于代码的 Transformer 单头注意力机制详解

## 整体流程图

```
输入 x (B, T, C=384)
   │
   ├──────────────────┬──────────────────┐
   ▼                  ▼                  ▼
  key(x)          query(x)           value(x)
   │                  │                  │
   ▼                  ▼                  ▼
   K (B,T,48)       Q (B,T,48)        V (B,T,48)
   │                  │                  │
   └────────▶ Q@K^T = 注意力分数 ◀────┘
                │
                ▼ /√d (缩放)
                ▼ softmax (归一化)
                ▼ dropout (随机丢弃)
                ▼
            注意力权重 (B,T,T)
                │
                ▼ @ V
                ▼
           输出 out (B,T,48)
```

---

## 代码逐行解析（第81-121行）

### 第一步：生成 Q、K、V（第101-102行）

```python
k = self.key(x)     # (B,T,48) ← 从输入提取"特征标签"
q = self.query(x)   # (B,T,48) ← 从输入提取"查询需求"
v = self.value(x)   # (B,T,48) ← 从输入提取"实际内容"
```

**通俗理解**：
- x 进来后，复制三份
- 三个线性层各自"问"x 不同的东西，得到三个视角

---

### 第二步：计算相似度（第107行）

```python
wei = q @ k.transpose(-2,-1) * (k.shape[-1] ** -0.5)
```

**详细拆解**：

```
q (B,T,48) @ k.transpose (B,48,T) = (B,T,T)

        k的第0个token  k的第1个token  k的第2个token ...
q的第0个token  [点积→相似度  点积→相似度  点积→相似度  ...]
q的第1个token  [点积→相似度  点积→相似度  点积→相似度  ...]
q的第2个token  [点积→相似度  点积→相似度  点积→相似度  ...]
...
```

**除以 √d（`k.shape[-1] ** -0.5` 即 48^-0.5 ≈ 0.144）**：
- d=48 越大，点积越大
- 除以后，值回到合理范围
- 防止 softmax 变成"硬焦点"（只关注一个位置）

---

### 第三步：掩码（第111行）

```python
wei = wei.masked_fill(self.tril[:T, :T] == 0, float("-inf"))
```

**作用**：下三角保留，上三角变 `-inf`

```
原本 (T=4):
[?, ?, ?, ?]
[?, ?, ?, ?]
[?, ?, ?, ?]
[?, ?, ?, ?]

掩码后:
[?, -, -, -]   # 第0个token：只能看自己
[?, ?, -, -]   # 第1个token：能看0,1
[?, ?, ?, -]   # 第2个token：能看0,1,2
[?, ?, ?, ?]   # 第3个token：能看0,1,2,3

# - = -inf
```

**注意**：代码使用 `self.tril[:T, :T]` 动态切片，适配生成过程中序列长度小于 block_size 的情况。

**目的**：预测第 T 个字时，只能用前 T-1 个字，不能"偷看"答案

---

### 第四步：归一化（第114行）

```python
wei = F.softmax(wei, dim=-1)
```

**作用**：把相似度变成"概率分布"（每行和为1）

```
softmax前:
[3.0, 1.0, 0.5]

softmax后:
[0.88, 0.12, 0.00]  # 越相关的越受关注
```

---

### 第五步：dropout（第115行）

```python
wei = self.dropout(wei)
```

随机把一些注意力权重置零，防止过拟合

---

### 第六步：加权求和（第118-119行）

```python
v = self.value(x)    # 提取value
out = wei @ v        # 加权求和
```

**核心公式**：`out = 注意力权重 × V`

```
第0个token的输出 = 
  第0个token的注意力权重 × V[0] +
  第1个token的注意力权重 × V[1] +
  第2个token的注意力权重 × V[2] + ...
```

---

## 完整流程举例

假设输入是 "今天天气"（T=4）：

```
第0个token"今"关注自己 80% + "天" 10% + "气" 5% + "天" 5%
第1个token"天"关注"今" 30% + 自己 50% + "气" 15% + "天" 5%
...
```

每个 token 都去"问"前面的 token：
- Q："我需要什么信息？"
- K："我有什么信息？"
- 匹配 → 得到权重 → 从 V 中取值 → 输出

## 总结

| 步骤 | 代码 | 作用 |
|------|------|------|
| 1 | `q @ k.T` | 计算所有位置两两之间的相关性 |
| 2 | `/√d` | 缩放，防止数值过大 |
| 3 | `masked_fill` | 因果掩码，只看上文 |
| 4 | `softmax` | 变成概率分布 |
| 5 | `dropout` | 随机丢弃，增强泛化 |
| 6 | `@ v` | 加权求和，得到输出 |

## Q、K、V 的本质

| 概念 | 作用 | 你的例子 |
|------|------|----------|
| **Q (Query)** | 我现在需要什么 | "我想找科幻小说" |
| **K (Key)** | 每个位置有什么特征 | 书架上每本书的分类标签 |
| **V (Value)** | 实际的信息内容 | 书架上书的具体内容 |

**关键点**：
- Q、K、V 是三个**可学习的线性变换**，从同一个输入 X 变出来
- 本质是**三个不同的视角**去看同一个东西
- 不是必须这样设计，但这样设计效果最好，网络可以自由调整每个变换学到什么特征

## 为什么要除以 √d ？

### d 的含义
`d` = Q（或 K）向量的维度，即 `head_size = 48`

### 问题：点积的方差会随 d 增大
假设 Q、K 每个元素是标准正态分布（均值0，方差1）：
- 单个元素的方差 = 1
- 点积 `Q·K = q1*k1 + q2*k2 + ... + qd*kd`
- d 个独立变量相加 → **方差 = d**

**d 越大，点积结果越大、波动越剧烈**

### 危害：softmax 梯度消失
softmax 函数的特点：
- 输入值**小** → 输出接近均匀分布
- 输入值**大** → 输出趋近于 one-hot（只关注一个位置）

```
d=256  →  点积可能很大  →  softmax 几乎变成 one-hot  →  梯度≈0  →  训练不动
```

### 解决：除以 √d
```python
wei = (Q @ K^T) / √d  # 缩放到合理范围
```
- 除以 √d 后，**方差 = 1**
- 点积结果回到合理区间
- softmax 输出分布稳定，梯度正常

**一句话**：`√d` 是为了**稳定训练**，防止点积过大让 softmax "硬化"，导致梯度消失。
